## Overview

Ai Studio includes two AI-powered assistants that handle customer orders and reservations automatically: **Slice** (web chat) and the **WhatsApp Bot**. Both use Google's Gemini AI with custom prompts that include real-time menu data, availability, and business rules.

### Key Benefits

- **24/7 Automated Service**: Accept orders and reservations even when closed
- **Multi-Channel Support**: Web interface and WhatsApp for customer convenience
- **Context-Aware**: AI knows menu, prices, availability, and business hours
- **Intelligent Conversation**: Natural language processing for Spanish interactions
- **Order Validation**: Enforces payment rules, availability checks, and time restrictions
- **Metrics Tracking**: Monitor conversations, token usage, and conversion rates

export const getMetrics = (): SliceBotMetrics => {
  const sessions = getAllSessions();
  
  return {
    distinctCustomers: sessions.length,  // Unique sessions
    totalMessages: sessions.reduce((sum, s) => sum + s.messages.length, 0),
    totalTokensUsed: sessions.reduce((sum, s) => sum + s.tokensUsed, 0),
    ordersMade: sessions.filter(s => s.outcome === 'order').length,
    reservationsMade: sessions.filter(s => s.outcome === 'reservation').length,
  };
};

## Troubleshooting

<Accordion title="AI Creates Order But It Doesn't Appear in Admin">
  **Possible Causes**:
  - Firebase write failed (check network/permissions)
  - Order validation failed (check console for errors)
  
  **Solution**:
  - Check browser console for error messages
  - Verify Firebase security rules allow writes
  - Test with manual order creation in admin panel
</Accordion>

<Accordion title="WhatsApp Bot Shows 'Error' Status">
  **Possible Causes**:
  - BuilderBot API credentials invalid
  - Network connectivity issue
  - WhatsApp phone disconnected
  
  **Solution**:
  - Disconnect and reconnect bot
  - Verify API keys in `whatsappBotService.ts`
  - Check BuilderBot dashboard for status
</Accordion>

<Accordion title="AI Suggests Unavailable Reservation Time">
  **Possible Causes**:
  - Real-time availability data not loading
  - Race condition (table booked between AI suggestion and submission)
  
  **Solution**:
  - System performs final availability check before creating reservation
  - If unavailable, AI offers alternatives automatically
  - This is expected behavior for concurrent bookings
</Accordion>

<Accordion title="Voice Input Not Working">
  **Possible Causes**:
  - Microphone permissions denied
  - Browser doesn't support MediaRecorder API
  - Network issue during transcription
  
  **Solution**:
  - Check browser permissions for microphone
  - Use Chrome/Edge (best support)
  - Verify Gemini API key is valid
</Accordion>

### Gemini Integration

**Model**: `gemini-2.5-flash`

**Use Cases**:

- JSON structured output

**Pricing Considerations**:

### BuilderBot Integration

**API**: BuilderBot Cloud REST API  
**Version**: v1 (deploy), v2 (messages)  
**Endpoints**:
- `POST /api/v1/manager/deploys` - Start bot
- `GET /api/v1/manager/deploys/{id}/qr` - Get QR code
- `GET /api/v1/manager/deploys/{id}` - Check status
- `DELETE /api/v1/manager/deploys/{id}` - Stop bot
- `POST /api/v2/{projectId}/messages` - Send message

**Authentication**: `x-api-builderbot` header with API key
